## Что сделано в последней сессии

- **Ингредиенты снова видны:** в схеме БД нет колонки `show_in_selector`, а код её запрашивал и фильтровал по ней → запрос падал, список был пустой. В `src/services/ingredients.ts` убрали `show_in_selector` из SELECT, в маппинге ставим `show_in_selector: true` для всех.
- **Поиск блюд только из выбранного:** раньше овощи не проверялись («могут быть гарниром»), поэтому показывался, например, «Картофель с яичницей» без картофеля в выборе. В `src/services/dishes.ts` убрали исключение для овощей — теперь проверяются **все** ингредиенты блюда; блюдо показывается только если каждый его ингредиент входит в выбранные + базовые (соль, перец, масла). При выключенном «Немного докупить» (allowMissing: 0) — строго только из выбранного.
- Ранее: деплой на Vercel, HEIC (heic2any), правка/добавление продуктов после фото, Колбаса слайсами и яичница/омлет в базе, ИИ-подсказки при 0 блюд, handoff (PROGRESS.md + правило).
- **Сессия 2026-03-09:** Анализ архитектуры тестирования. Составлен план внедрения QA-агента (Playwright E2E + Vitest unit + DB audit). Подробности — в разделе «Следующие шаги».

## Архитектура "агентов" в проекте

Формальных агентов нет — приложение браузерное, без бэкенда. Условные "агенты":

| Условный агент | Что делает | Файлы |
|---|---|---|
| **OpenAI-агент** | Анализирует фото холодильника, определяет ингредиенты, считает калории, предлагает блюда когда БД не дала результат | `src/services/openai.ts`, `src/store/slices/photoSlice.ts` |
| **DB-агент (sql.js)** | Поиск блюд по ингредиентам, фильтрация, рецепты | `src/services/dishes.ts`, `src/services/ingredients.ts`, `src/services/recipes.ts` |
| **Redux state machine** | Координирует состояние между компонентами (selectedIngredients → findDishes → recipe) | `src/store/slices/` |
| **Handoff-агент (PROGRESS.md)** | Синхронизация контекста Claude Code ↔ Cursor | `docs/PROGRESS.md`, `.cursor/rules/handoff-sync.mdc` |

**Синхронизация между агентами:**
- `PhotoUpload` → `analyzeIngredients` (OpenAI) → `setSelectedIngredients` (Redux) → `findDishesByIngredients` (sql.js)
- Цепочка: фото → имена ингредиентов → ID из БД → поиск блюд по ID → рецепт

## Следующие шаги — ПЛАН QA-АГЕНТА

### Подход: 3 уровня тестирования

#### Уровень 1: DB Audit Script (быстро, без браузера)
**Файл:** `scripts/audit-db.ts`
Проверяет целостность данных:
- Все блюда имеют ≥1 ингредиент в `dish_ingredients`
- Все блюда имеют рецепт в `recipes`
- Все ингредиенты рецепта есть в `ingredients`
- Нет блюд без `cooking_time` или `difficulty`
- Каждый dish_ingredient ID реально существует в `ingredients`

Запуск: `npx tsx scripts/audit-db.ts`

#### Уровень 2: Vitest Unit Tests (бизнес-логика)
**Файлы:** `tests/unit/dishes.test.ts`, `tests/unit/ingredients.test.ts`
Тестирует без браузера:
- `findDishesByIngredients([яйца_id, помидоры_id])` → не должен возвращать плов
- `findDishesByIngredients([яйца_id, помидоры_id])` → должен возвращать яичницу/омлет
- allowMissing логика
- RandomizerFocus фильтрация

Запуск: `npx vitest run`

#### Уровень 3: Playwright E2E (полный флоу с браузером)
**Файлы:** `tests/e2e/photo-flow.spec.ts`, `tests/e2e/dish-search.spec.ts`
Тестирует реальный UI:
- Загружает тестовое фото (fixtures/fridge-eggs-tomatoes.jpg)
- Проверяет что чипы с ингредиентами появились
- Кликает «Найти блюда»
- Проверяет что в результатах есть яичница/омлет, но НЕТ плова
- Кликает на блюдо → проверяет что рецепт открылся

Моки OpenAI: перехватывает `fetch` к api.openai.com, возвращает фиксированный ответ

Запуск: `npx playwright test`
