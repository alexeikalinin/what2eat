# What2Eat (ЧтоЕсть)

Приложение для **планирования приёмов пищи** для семьи (например, семейной пары). Помогает решить вопрос «что приготовить», опираясь на имеющиеся ингредиенты или случайный выбор блюд.

## Для чего проект

- **Выбор по ингредиентам** — отмечаете, какие продукты есть дома, приложение подбирает подходящие блюда.
- **Рандомайзер** — кнопка «Рандомайзер блюд» выбирает мясные блюда случайным образом, когда не хочется думать.
- **Рецепты** — у каждого блюда есть рецепт с пошаговыми инструкциями и количеством ингредиентов.

## Стек

- **Frontend:** React 18, TypeScript, Vite
- **UI:** Material UI (MUI)
- **Состояние:** Redux Toolkit
- **Данные:** SQLite в браузере (sql.js), схема и начальные данные в `public/database/`

## Структура данных

- **ingredients** — ингредиенты по категориям (мясо, крупы, овощи, молочное, специи и т.д.)
- **dishes** — блюда (название, описание, время, сложность, порции)
- **dish_ingredients** — связь блюд и ингредиентов (многие-ко-многим)
- **recipes** — пошаговые инструкции (JSON)
- **recipe_ingredients** — ингредиенты рецепта с количеством и единицами (г, мл, шт и т.д.)

## Запуск

```bash
npm install   # если ещё не установлено
npm run dev   # dev-сервер на http://localhost:3000
```

Сборка и превью:

```bash
npm run build
npm run preview
```

## Скрипты

| Команда    | Описание                    |
|-----------|-----------------------------|
| `npm run dev`     | Запуск dev-сервера (Vite)   |
| `npm run build`   | Сборка для продакшена       |
| `npm run preview` | Просмотр собранного билда   |
| `npm run lint`    | Проверка ESLint             |
| `npm run format`  | Форматирование Prettier     |

База данных инициализируется при первом открытии приложения: загружаются `schema.sql` и `seed.sql` из `public/database/`.

## Деплой на Vercel

1. Импортируйте репозиторий в [Vercel](https://vercel.com) (GitHub → New Project).
2. Сборка и выходная папка заданы в `vercel.json` (`npm run build`, `dist`). Менять ничего не нужно.
3. (Опционально) Для распознавания ингредиентов по фото добавьте переменную окружения в **Settings → Environment Variables**: `VITE_OPENAI_API_KEY` со значением вашего OpenAI API ключа.
4. Деплой: каждый push в `main` собирает и публикует приложение.
